### 1. 本日の作業概要

[cite_start]イベント「デジタライブ・アドベンチャー」で使用するNFCカードの初期設定（参加者情報書き込み）を行うためのGUIアプリケーション（Electron）の開発環境構築を実施しました。 [cite: 3, 6, 8]
[cite_start]主な作業は、NFCリーダー制御モジュール `nfc-pcsc` のインストール時に発生した `node-gyp` （C++コンパイル）エラーの解決です。 [cite: 10, 20]

### 2. 主な変更点（環境構築）

今回はコードやドキュメントの変更ではなく、開発PCの環境構築を行いました。

* **インストールしたツール（PC環境）:**
    * [cite_start]**Build Tools for Visual Studio 2022** [cite: 26]
        * [cite_start]目的: `nfc-pcsc` などC++コードを含むNode.jsモジュールをWindows上でコンパイルするためにインストールしました。 [cite: 27]
        * [cite_start]詳細: インストーラーで「C++によるデスクトップ開発」ワークロードを追加しました。 [cite: 28]

* **インストールしたモジュール（Node.js / npm）:**
    * [cite_start]**`nfc-pcsc`** [cite: 23]
        * [cite_start]目的: ElectronからACR122U（NFCリーダー）を制御するための中核ライブラリとしてインストールしました。 [cite: 7, 24]

### 3. 技術的なメモ・課題

* **発生した課題:**
    * [cite_start]`npm install nfc-pcsc` 実行時に、`node-gyp` エラー（C++コンパイルエラー）が多発しました。 [cite: 10]

* **特定した原因:**
    * [cite_start]PCにC++ビルドツール（コンパイラ）がインストールされていませんでした。 [cite: 12]
    * [cite_start]PCにインストールされていた Visual Studio "Insiders"版（プレビュー版）が、npmのコンパイル処理を妨害していました。 [cite: 13]
    * [cite_start]PowerShellの実行ポリシーにより、npm関連のスクリプトがブロックされていました。 [cite: 14]

* **最終的な解決策:**
    * [cite_start]"Insiders" ではない、安定版の「Developer Command Prompt for VS 2022」を起動しました。 [cite: 16]
    * [cite_start]この専用プロンプト内で `npm install nfc-pcsc` を実行することで、コンパイルに成功しました。 [cite: 17, 20]
